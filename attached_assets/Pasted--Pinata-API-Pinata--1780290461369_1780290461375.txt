أهلاً أختي مريم، أسئلتك في محلها، وخاصة نقطة الـ Pinata API مهمة جداً.

نعم، باش تخدمي مع **Pinata** (أو أي خدمة IPFS أخرى)، خاصك ديري حساب وتجيبي **مفاتيح API** (API keys) باش الـ Agent يقدر يرفع الصور ويخزنها. هاد الخدمة كاتعطيك 1 جيجا مجانا، و كاتزاد بعد ذلك بأسعار رمزية.

### 🔧 شلون تجيبي مفتاح API من Pinata؟

1.  **سجّلي الدخول** في موقع Pinata: [https://app.pinata.cloud/](https://app.pinata.cloud/).
2.  **دوري على قسم "API Keys"** في لوحة التحكم، عادة ما يكون تحت `Developers`.
3.  **دوسي على "New Key"** باش تولدي مفتاح جديد.
4.  **أعطي المفتاح إسماً**، مثلاً `FrostDex-Key`، و أعطيه الصلاحيات المطلوبة: **Admin privileges و unlimited uses** (صلاحية `pinFileToIPFS` كافية باش ترفعي الصور).
5.  **خدي الـ JWT token**، هو سلسلة طويلة كاتبدا بـ `eyJ...`، و خليها فمكان آمن.

### 💬 شنو نديرو بهاد المفتاح؟

بعد ما تجيبي الـ JWT، هذا هو **الـ Prompt اللي تعطيه لـ Replit AI Agent**، و هو كايتكفل بكل شيء:

*   **الإعداد**: يركب المكتبة وينشئ ملفات الخدمة.
*   **الربط**: يجهز الكود باش يستعمل الـ API.
*   **المتغير**: يطلب منك تدخلي المفتاح اللي نسختيه بطريقة آمنة.
*   **الفورم**: يضيف جزء رفع الصورة للفورم بشكل صحيح.
*   **الـ Program ID**: يشرح ليك كيفاش تولديه من محفظتك مباشرة (طريقة جديدة).

### ✍️ الـ Prompt الكامل لـ AI Agent (انسخي و الصقي)

```markdown
أريد إضافة ميزة رفع الصور إلى IPFS عبر Pinata في مشروع Vite + React الخاص بي (الموجود في Replit). أرجو منك إتمام المهام التالية بالترتيب:

1.  **ثبّت المكتبة المطلوبة**:
    ```bash
    npm install pinata-web3
    ```

2.  **أنشئ ملف الخدمة** `src/services/ipfs.ts`، مع التأكد من استعمال أحدث SDK من Pinata (pinata-web3):
    ```typescript
    import { PinataSDK } from 'pinata-web3';

    const pinata = new PinataSDK({
      pinataJwt: import.meta.env.VITE_PINATA_JWT,
      pinataGateway: 'gateway.pinata.cloud',
    });

    export async function uploadImageToIPFS(file: File): Promise<string> {
      const result = await pinata.upload.file(file);
      return `https://gateway.pinata.cloud/ipfs/${result.IpfsHash}`;
    }

    export async function uploadJSONToIPFS(jsonData: object): Promise<string> {
      const result = await pinata.upload.json(jsonData);
      return `https://gateway.pinata.cloud/ipfs/${result.IpfsHash}`;
    }
    ```

3.  **أضف متغير البيئة `VITE_PINATA_JWT`**:
    - اطلب مني (أنا المستخدم) أن أذهب إلى علامة التبويب "Secrets" في Replit (أيقونة القفل). سأقوم هناك بإنشاء مفتاح جديد باسم `VITE_PINATA_JWT` وسألصق فيه مفتاح الـ JWT الخاص بي (الذي حصلت عليه من موقع Pinata).
    - لا تقم بتخزين المفتاح في ملف `.env` بنفسك، فقط استخدم `import.meta.env.VITE_PINATA_JWT` في الكود.

4.  **دمج الـ API في باقي الكود**:
    - استعمل دالة `uploadImageToIPFS` في صفحة إنشاء التوكن (`CreateToken.tsx`) لرفع صورة التوكن.
    - استعمل دالة `uploadJSONToIPFS` لرفع البيانات الوصفية بعد رفع الصورة.

5.  **إضافة خيار رفع الصورة في الفورم**:
    - تأكد من أن صفحة `/create-token` تحتوي على جزء لرفع الصورة (Image Upload) كما هو موضح في التصميم السابق.

6.  **بخصوص الـ Program ID**:
    - لما نحتاجو للـ `Program ID`، غادي نتبع هاد الطريقة باش نولّدوه مباشرة من محفظتي. سأقوم بربط محفظتي بـ Phantom من خلال الواجهة التي ستبنيها، والمطلوب منك هو أن تضمن أن الكود جاهز ليقرأ هذا العنوان بعد ربط المحفظة. أنا شخصياً سأقوم بتأكيد المعاملة فقط.
    - لا تتطلب مني أي شيء آخر، فقط أنتظر مني أن أقول لك أني قمت بربط المحفظة.

ابدأ الآن بتنفيذ هذه المهام.
```

بعد ما تعطي هذا الـ prompt للـ Agent، هو غادي يركب المكتبات و ينشئ الملفات و يعطيك التعليمات. أنت فقط غادي تروحي لـ "Secrets" و تزيدي المفتاح. بالنسبة لـ **Program ID**، هو وليد محفظتك. بعد ما تشتغلي الواجهة و تكوني مقسمة (connected)، غادي يبان ليك عنوان العقد تلقائياً.
